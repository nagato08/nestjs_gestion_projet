# Module IA – Tests

Interprétation en langage naturel (créer tâche, assigner), exécution d’actions, analyse Gantt/PERT, prévision des retards.

---

## Prérequis

- **JWT** pour toutes les routes.
- **ADMIN** ou **chef de projet (owner)** du projet : toutes les routes IA sont protégées par `AiAllowedGuard`.
- **OPENAI_API_KEY** définie dans `.env`, sinon les appels renvoient une erreur.

---

## Endpoints

### 1. Interpréter un message (suggestion sans exécution)

| Élément | Valeur                         |
| ------- | ------------------------------ |
| Méthode | `POST`                         |
| URL     | `/ai/interpret`                |
| Auth    | Oui + Admin ou owner du projet |

**Body (JSON)** : `projectId`, `message` (demande en langage naturel, 3–2000 caractères).

**Exemple curl :**

```bash
curl -s -X POST http://localhost:4000/ai/interpret \
  -H "Authorization: Bearer VOTRE_JWT" \
  -H "Content-Type: application/json" \
  -d '{
    "projectId": "PROJECT_ID",
    "message": "Crée une tâche Revue design priorité haute et assigne-la à Marie"
  }'
```

**À vérifier :** `action` (create_task | assign_task | null), `params`, `summary`, `confidence`. Si aucune action reconnue : `action: null`, `params: null`.

---

### 2. Exécuter une action suggérée

| Élément | Valeur               |
| ------- | -------------------- |
| Méthode | `POST`               |
| URL     | `/ai/execute`        |
| Auth    | Oui + Admin ou owner |

**Body (JSON)** : `projectId`, `action` ("create_task" | "assign_task"), `params` selon l’action :

- **create_task** : `title`, `priority` (HIGH | MEDIUM | LOW), optionnel : `description`, `assigneeId`
- **assign_task** : `taskId`, `userId`

**Exemple curl (créer tâche) :**

```bash
curl -s -X POST http://localhost:4000/ai/execute \
  -H "Authorization: Bearer VOTRE_JWT" \
  -H "Content-Type: application/json" \
  -d '{
    "projectId": "PROJECT_ID",
    "action": "create_task",
    "params": {
      "title": "Tests E2E",
      "priority": "HIGH",
      "description": "Couverture des parcours critiques",
      "assigneeId": "USER_ID"
    }
  }'
```

**Exemple curl (assigner) :**

```bash
curl -s -X POST http://localhost:4000/ai/execute \
  -H "Authorization: Bearer VOTRE_JWT" \
  -H "Content-Type: application/json" \
  -d '{
    "projectId": "PROJECT_ID",
    "action": "assign_task",
    "params": { "taskId": "TASK_ID", "userId": "USER_ID" }
  }'
```

**À vérifier :** Résultat de l’action (tâche créée ou assignations). 403 si non-admin et non-owner.

### 3. Interpréter puis exécuter (flux « act »)

| Élément | Valeur               |
| ------- | -------------------- |
| Méthode | `POST`               |
| URL     | `/ai/act`            |
| Auth    | Oui + Admin ou owner |

**Body (JSON)** : `projectId`, `message`.

**Exemple curl :**

```bash
curl -s -X POST http://localhost:4000/ai/act \
  -H "Authorization: Bearer VOTRE_JWT" \
  -H "Content-Type: application/json" \
  -d '{
    "projectId": "PROJECT_ID",
    "message": "Crée la tâche Déploiement staging priorité haute"
  }'
```

**À vérifier :** `executed` (true/false), `summary`, `action`, `data` (résultat si exécuté). Si l’IA ne propose pas d’action : `executed: false`, `data: null`.

### 4. Interprétation du diagramme de Gantt

| Élément | Valeur                         |
| ------- | ------------------------------ |
| Méthode | `GET`                          |
| URL     | `/ai/analyze/gantt/:projectId` |
| Auth    | Oui + Admin ou owner           |

**Exemple curl :**

```bash
curl -s "http://localhost:4000/ai/analyze/gantt/PROJECT_ID" \
  -H "Authorization: Bearer VOTRE_JWT"
```

**À vérifier :** `analysis` (texte en français : enchaînements, risques, charge, recommandations).
